# Heat Predictor Trace Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `dev` 增加默认关闭、不会阻塞 I/O、可由 MGR 控制并可离线转换的已评估 I/O Trace。

**Architecture:** `HpTraceWriter` 封装固定二进制 schema、有界队列和后台 writer；
`HeatPredictor` 在评估完成或异常终止点提交记录；OSD/MGR 只负责生命周期控制和状态输出。

**Tech Stack:** C++17、Ceph admin socket/MGR command、Python 3 CSV 转换器、现有算法探针。

## Global Constraints

- 只在 `dev` 实施，默认关闭。
- 前台使用有界 MPSC ring；只在 ring 满时丢 trace，不得等待 writer 或磁盘。
- reset/enable/disable 轮换 session；正常与异常样本必须可区分。
- 当前任务不提交或推送 Trace 改动。
